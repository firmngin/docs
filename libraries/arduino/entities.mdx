Entities are the user-facing values and controls exposed by a device. Use them for sensor readings, relay state, status values, counters, and dashboard commands.

[image:device entity list with temperature, relay, status, and command controls]

## Define entities

Use an `Entity` object when you want a typed reference to an entity key.

```cpp
Entity relay1("gpio_1");
Entity relay2("relay_2");
Entity temperature("temperature");
```

You can also use string keys directly when the sketch is small.

## Push state

Use `pushEntity()` for a single value update. This is useful for sensors, relay feedback, or any state that changes over time.

```cpp
void loop() {
  fngin.loop();

  static unsigned long lastPush = 0;
  if (millis() - lastPush >= 10000) {
    lastPush = millis();
    fngin.pushEntity("temperature", "25.5");
    fngin.pushEntity(relay1, "1");
  }
}
```

For multiple values at once, use the batch builder.

```cpp
bool sent = fngin.pushBatchEntities()
  .add("temperature", "25.5")
  .add("humidity", "80")
  .add("pressure", "1013")
  .add("light", String(analogRead(A0)))
  .add("status", "ONLINE")
  .send();

if (sent) {
  Serial.println("Batch state sent");
}
```

## Receive commands

Use entity callbacks to receive commands sent from Firmngin to the device. The command includes an entity key and value, so the firmware can route it to the correct GPIO or component.

```cpp
Entity relay1("gpio_1");

ON_ENTITY(relay1, [](EntityCommand &cmd) {
  digitalWrite(2, cmd.value() == "1" ? HIGH : LOW);
});
```

For multiple entity commands, register one global entity command callback and switch by key.

```cpp
fngin.on(ENTITIES, [](EntityCommand &cmd) {
  if (cmd.key() == "gpio_1") {
    digitalWrite(2, cmd.value() == "1" ? HIGH : LOW);
  }

  if (cmd.key() == "relay_2") {
    digitalWrite(4, cmd.value() == "1" ? HIGH : LOW);
  }
});
```

## Request init

Call `requestInit()` when the firmware needs the latest device configuration after connection or reconnect.

```cpp
void setup() {
  // Connect Wi-Fi first.
  fngin.begin();
  fngin.requestInit();
}
```

## Typed callbacks

The Arduino library supports typed callback objects for common flows.

| Callback type | Use it for |
| --- | --- |
| `EntityCommand` | Entity commands from the dashboard or automation. |
| `DeviceStates` | Device lifecycle and monetization state changes. |
| `Inits` | Initial entity, merchant, and verification configuration. |
| `Payments` | Pending and successful payment events. |
| `Verifications` | PIN display and verification result events. |
| `Usages` | Usage limit state. |
| `ActiveSession` | Device-side checks for an active paid session. |

Use the typed objects instead of parsing raw payloads unless you specifically need raw metadata.

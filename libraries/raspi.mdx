Gunakan Firmngin Python SDK untuk Raspberry Pi, Linux-based devices, integration tests, dan server-side tooling.

## Client vs AsyncClient

Dua entry point, satu protokol — pilih sesuai gaya kode Anda:

| | `AsyncClient` | `Client` |
| --- | --- | --- |
| Gaya pemrograman | `async` / `await` | Pemanggilan biasa |
| Cocok untuk | App asyncio, service berjalan terus | Skrip sederhana, cron, CLI |
| Daftar handler | `client.on(...)`, `client.on_entity(...)` | Sama |
| Kirim state | `await client.push_entity(...)` | `client.push_entity(...)` |
| Terima event | `await client.run()` | `client.run()` |

Keduanya memakai `ClientConfig.from_file("keys.json")` dan mendukung event yang sama.

## Features

- TLS, mTLS, dan certificate fingerprint validation
- E2EE (AES-GCM) aktif secara default
- Typed payloads, entity publishing, batch, location, offline queue
- Image upload opsional via `firmngin[http]`

## Requirements

- Python 3.9+
- File `keys.json` dari dashboard Firmngin

## Dependencies

| Package | Fungsi |
| --- | --- |
| `aiomqtt` | Transport MQTT |
| `cryptography` | TLS + enkripsi E2EE |

Upload gambar: `pip install firmngin[http]`

## Quickstart

**Async** — untuk app berbasis `asyncio`:

```python
import asyncio
from firmngin import AsyncClient, ClientConfig, Entity

async def main():
    config = ClientConfig.from_file("keys.json")
    async with AsyncClient(config) as client:
        await client.connect()
        await client.push_entity(Entity("relay"), "on")
        await client.run()

asyncio.run(main())
```

**Sync** — untuk skrip tanpa `asyncio`:

```python
from firmngin import Client, ClientConfig, Entity

with Client(ClientConfig.from_file("keys.json")) as client:
    client.connect()
    client.push_entity(Entity("relay"), "on")
    client.run()
```

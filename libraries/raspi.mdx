Use the Firmngin Python SDK on Raspberry Pi, Linux-based devices, integration tests, and server-side tooling.

## Client vs AsyncClient

Two entry points, one protocol — choose what fits your code:

| | `AsyncClient` | `Client` |
| --- | --- | --- |
| Style | `async` / `await` | Regular blocking calls |
| Best for | Asyncio apps, long-running services | Simple scripts, cron jobs, CLI tools |
| Handlers | `client.on(...)`, `client.on_entity(...)` | Same |
| Push state | `await client.push_entity(...)` | `client.push_entity(...)` |
| Event loop | `await client.run()` | `client.run()` |

Both use `ClientConfig.from_file("keys.json")` and support the same events.

## Features

- TLS, mTLS, and certificate fingerprint validation
- E2EE (AES-GCM) enabled by default
- Typed payloads, entity publishing, batch updates, location, offline queue
- Optional image upload via `firmngin[http]`

## Requirements

- Python 3.9+
- A `keys.json` file from the Firmngin dashboard

## Dependencies

| Package | Purpose |
| --- | --- |
| `aiomqtt` | MQTT transport |
| `cryptography` | TLS + E2EE encryption |

Image upload: `pip install firmngin[http]`

## Quickstart

**Async** — for asyncio-based apps:

```python
import asyncio
from firmngin import AsyncClient, ClientConfig, Entity

async def main():
    config = ClientConfig.from_file("keys.json")
    async with AsyncClient(config) as client:
        await client.connect()
        await client.push_entity(Entity("relay"), "on")
        await client.run()

asyncio.run(main())
```

**Sync** — for scripts without asyncio:

```python
from firmngin import Client, ClientConfig, Entity

with Client(ClientConfig.from_file("keys.json")) as client:
    client.connect()
    client.push_entity(Entity("relay"), "on")
    client.run()
```

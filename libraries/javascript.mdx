Use the Firmngin JavaScript SDK on Node.js gateways, simulators, integration tests, and server-side tooling.

## Client vs AsyncClient

Two entry points, one protocol — choose what fits your code:

| | `AsyncClient` | `Client` |
| --- | --- | --- |
| Style | `async` / `await` | Same Promise-based API |
| Handlers | `client.on(...)`, `client.onEntity(...)` | Same |
| Push state | `await client.pushEntity(...)` | `await client.pushEntity(...)` |
| Event loop | `await client.run()` | `await client.run()` |

Both use `ClientConfig.fromFile("keys.json")` and support the same events.

## Features

- TLS, mTLS, and certificate validation (including dashboard `keys.json`)
- E2EE (AES-GCM) enabled by default
- LWT online/offline on connect and disconnect
- Auto `requestInit()` on connect
- Typed events, entity publishing, batch updates, and location helpers
- Payment metadata callbacks (`mop` / `moe` / `mos`) as raw JSON strings
- Image upload included in the main package

OTA firmware updates are not available in the JavaScript SDK.

## Requirements

- Node.js 18+
- A `keys.json` file from the Firmngin dashboard

## Dependencies

| Package | Purpose |
| --- | --- |
| `mqtt` | MQTT transport |
| `@noble/ciphers` | AES-GCM E2EE |

## Quickstart

**AsyncClient** — recommended for most apps:

```typescript
import { AsyncClient, ClientConfig, Entity } from 'firmngin'

const config = await ClientConfig.fromFile('keys.json')
const client = new AsyncClient(config)

await client.connect()
await client.pushEntity(new Entity('relay'), 'on')
await client.run()
```

**Client** — alternative entry point with the same API:

```typescript
import { Client, ClientConfig, Entity } from 'firmngin'

const client = new Client(await ClientConfig.fromFile('keys.json'))

await client.connect()
await client.pushEntity(new Entity('relay'), 'on')
await client.run()
```

## Debug mode

Debug is off by default. Enable it before `connect()` to print the startup banner and connection logs:

```typescript
client.setDebug(true)
await client.connect()
```

## Browser entry

The package also exports `firmngin/browser` for bundlers that target the browser. Device connectivity in the browser requires WSS and platform-specific constraints — use the Node entry for full device client features today.

```typescript
import { AsyncClient } from 'firmngin/browser'
```

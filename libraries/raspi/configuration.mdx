SDK memakai dua model konfigurasi: `KeysConfig` untuk identitas perangkat, dan `ClientConfig` untuk perilaku runtime.

## Quick setup

```python
from firmngin import ClientConfig

config = ClientConfig.from_file("keys.json")
```

## KeysConfig

Muat kredensial dari `keys.json`:

```python
from firmngin import KeysConfig

keys = KeysConfig.from_file("keys.json")
```

### Fields

| Field | Type | Description |
| --- | --- | --- |
| `device_id` | `str` | Device identifier dari Firmngin |
| `device_key` | `str` | Authentication key |
| `decryptor` | `str` | Hex-encoded AES key untuk E2EE |
| `validation_mode` | `str` | `"ca"`, `"pin"`, atau `"both"` |
| `ca_cert` | `str \| None` | CA certificate PEM |
| `service_ca_cert` | `str` | Service CA certificate PEM |
| `client_cert` | `str \| None` | Client certificate untuk mTLS |
| `private_key` | `str \| None` | Private key untuk mTLS |
| `fingerprint_sha256` | `str \| None` | SHA-256 fingerprint untuk pin mode |

Property `decryptor_key` menyimpan byte key yang sudah di-parse (cache internal).

### Load from environment

```python
keys = KeysConfig.from_env(prefix="FIRMNGIN_")
```

Membaca `FIRMNGIN_DEVICE_ID`, `FIRMNGIN_DEVICE_KEY`, dan seterusnya.

## ClientConfig

```python
from firmngin import ClientConfig, KeysConfig

config = ClientConfig(
    keys=KeysConfig.from_file("keys.json"),
    queue_path="/var/lib/firmngin/queue",
)
```

### Fields

| Field | Default | Description |
| --- | --- | --- |
| `keys` | (required) | Instance `KeysConfig` |
| `api_base_url` | `https://api.firmngin.dev/api/v1` | Base URL HTTP API |
| `queue_path` | `None` | Path offline queue (aktifkan untuk persist saat offline) |
| `queue_max_size` | `500` | Batas jumlah pesan dalam queue |
| `queue_max_bytes` | `5242880` | Batas ukuran total payload queue (bytes) |
| `queue_drain_batch_size` | `50` | Pesan yang di-drain per siklus connect |
| `max_local_entity_values` | `256` | Batas cache nilai entity lokal |
| `connect_timeout_seconds` | `10.0` | Timeout koneksi MQTT |
| `keepalive_seconds` | `60` | MQTT keepalive |
| `reconnect_initial_delay_seconds` | `1.0` | Delay awal reconnect |
| `reconnect_max_delay_seconds` | `30.0` | Delay maksimum reconnect |
| `reconnect_max_attempts` | `None` | Batas percobaan reconnect (`None` = unlimited) |
| `insecure` | `False` | Lewati validasi TLS (tidak untuk production) |
| `mtls` | `True` | Mutual TLS — butuh `client_cert` + `private_key` |

Broker MQTT (`asia-jkt1.firmngin.dev:58884`) dikelola platform dan tidak dikonfigurasi oleh aplikasi.

## TLS modes

| Mode | Use case |
| --- | --- |
| `"ca"` | Validasi CA — direkomendasikan production |
| `"pin"` | Fingerprint pinning — ringan, perlu update saat cert berubah |
| `"both"` | Default — CA + fingerprint |

## Offline queue

```python
config = ClientConfig.from_file(
    "keys.json",
    queue_path="/var/lib/firmngin/queue",
)
```

Pesan di-queue saat MQTT offline dan di-drain bertahap saat reconnect.

## Debug logging

```python
async with Client(config) as client:
    client.set_debug(True)
    await client.connect()
```

## Firmware metadata

```python
client.set_firmware_info(
    version="1.0.0",
    target_board="RASPBERRY_PI",
    target_model="Raspberry Pi 4 Model B",
)
```

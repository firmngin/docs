This guide covers CI/CD for Arduino-style projects built with [Arduino CLI](https://arduino.github.io/arduino-cli/) — repositories that contain `.ino` sketch files and use FQBN (Fully Qualified Board Name) to define the target board.

The [`firmngin/ota`](https://github.com/firmngin/ota) action auto-detects Arduino projects when no `platformio.ini` is present and at least one `.ino` file exists within three directory levels of the repository root.

## When to use Arduino CLI CI/CD

| Scenario | Arduino CLI CI/CD fit |
| --- | --- |
| Standard Arduino sketch layout (`MyDevice/MyDevice.ino`) | Excellent |
| Arduino IDE projects migrated to git | Supported with FQBN |
| Sketches using `#include <firmngin.h>` | Supported — action installs Firmngin library automatically |
| PlatformIO project | Use [PlatformIO CI/CD](/firmware-ci-cd-platformio) instead |
| Multiple `.ino` files in one repo | Set `arduino-sketch` explicitly |

## Sketch folder layout (required)

Arduino CLI requires the **folder name and `.ino` filename to match**:

```text
✅ Correct
my-device/
  my-device.ino
  keys.h

❌ Wrong
my-device.ino          ← lone file at repo root
repo-name.ino          ← filename must match folder, not repo name
```

Set `arduino-sketch` to the **folder name**:

```yaml
arduino-sketch: my-device
```

## Project requirements

| Requirement | Details |
| --- | --- |
| Sketch folder | `Name/Name.ino` matching layout |
| FQBN | Valid board identifier (for example `esp32:esp32:esp32`) |
| `keys.h` | Beside the sketch for Firmngin library projects — **never commit to git** |
| firmngin board/model | Device or template fields match your FQBN target |

## Complete workflow example (single device)

Tested pattern for ESP32 + Firmngin library + single-device OTA:

```yaml
name: Firmngin OTA

on:
  push:
    branches:
      - main
  workflow_dispatch:

permissions:
  contents: read

jobs:
  ota:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Prepare keys.h
        env:
          KEYS_H: ${{ secrets.FIRMNGIN_KEYS_H }}
        run: |
          if [ -z "$KEYS_H" ]; then
            echo "Missing secret FIRMNGIN_KEYS_H" >&2
            exit 1
          fi
          printf '%s\n' "$KEYS_H" > ard-ci-test/keys.h

      - uses: firmngin/ota@v1.1.3
        with:
          token: ${{ secrets.FIRMNGIN_API_TOKEN }}
          device-secret-id: ${{ secrets.FIRMNGIN_DEVICE_SECRET_ID }}
          platform: arduino
          arduino-fqbn: esp32:esp32:esp32
          arduino-sketch: ard-ci-test
          version: ci-${{ github.run_number }}
          release-notes: ${{ github.event.head_commit.message }}
```

### GitHub Secrets required

| Secret | Value |
| --- | --- |
| `FIRMNGIN_API_TOKEN` | Token from **Integrations** (Firmware OTA scope) |
| `FIRMNGIN_KEYS_H` | Full contents of local `keys.h` (copy-paste entire file) |
| `FIRMNGIN_DEVICE_SECRET_ID` | Device **Secret ID** from dashboard (same as `DEVICE_ID` in `keys.h`) |

### `.gitignore`

```gitignore
keys.h
ard-ci-test/keys.h
```

## What the action does for Arduino builds

On each run, `firmngin/ota` automatically installs the board core, the Firmngin Arduino library, compiles your sketch, uploads the firmware, and triggers OTA when configured.

You do **not** need a separate step to install the Firmngin library.

## Find your FQBN

```bash
arduino-cli core update-index
arduino-cli board listall esp32
```

Common values:

| Hardware | FQBN |
| --- | --- |
| ESP32 Dev Module | `esp32:esp32:esp32` |
| ESP8266 NodeMCU | `esp8266:esp8266:nodemcuv2` |

## Fleet OTA via template

```yaml
- uses: firmngin/ota@v1.1.3
  with:
    token: ${{ secrets.FIRMNGIN_API_TOKEN }}
    device-template-id: ${{ vars.FIRMNGIN_DEVICE_TEMPLATE_ID }}
    platform: arduino
    arduino-fqbn: esp32:esp32:esp32
    arduino-sketch: my-device
```

Still inject `keys.h` in a prior step if your sketch uses the Firmngin library.

## Local verification

```bash
arduino-cli core update-index
arduino-cli core install esp32:esp32
git clone --depth 1 https://github.com/firmngin/Firmngin-Arduino.git ~/Arduino/libraries/Firmngin-Arduino
arduino-cli compile --fqbn esp32:esp32:esp32 my-device
find my-device -name '*.bin'
```

## Common errors

| Error | Cause | Fix |
| --- | --- | --- |
| `Can't open sketch: main file missing` | Invalid sketch layout | Use `Folder/Folder.ino`; set `arduino-sketch: Folder` |
| `device-secret-id ... is required` | Empty secret/variable | Add `FIRMNGIN_DEVICE_SECRET_ID` to **GitHub Secrets** |
| `Firmware binary not found after Arduino compile` | Build output not found | Upgrade to `firmngin/ota@v1.1.3` or newer |
| `Cloning into '.../Firmngin-Arduino'` in logs | Library install step | Normal — wait for the workflow to finish |
| Compile error: `firmngin.h` not found | Library missing locally | Clone Firmngin-Arduino locally or use CI action |
| Missing `keys.h` | File gitignored, not injected in CI | Add `FIRMNGIN_KEYS_H` secret + Prepare step |

## Security

- Never commit `keys.h` — contains device credentials and certificates.
- Store token, keys, and device Secret ID in **GitHub Secrets** only.
- Remove hardcoded WiFi passwords from sketches before pushing to public repos.

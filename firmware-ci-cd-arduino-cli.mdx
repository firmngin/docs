This guide covers CI/CD for Arduino-style projects built with [Arduino CLI](https://arduino.github.io/arduino-cli/) — repositories that contain `.ino` sketch files and use FQBN (Fully Qualified Board Name) to define the target board.

The [`firmngin/ota`](https://github.com/firmngin/ota) action auto-detects Arduino projects when no `platformio.ini` is present and at least one `.ino` file exists within three directory levels of the repository root.

## When to use Arduino CLI CI/CD

| Scenario | Arduino CLI CI/CD fit |
| --- | --- |
| Standard Arduino sketch layout (`MyDevice/MyDevice.ino`) | Excellent |
| Arduino IDE projects migrated to git | Supported with FQBN |
| PlatformIO project | Use [PlatformIO CI/CD](/firmware-ci-cd-platformio) instead |
| Multiple `.ino` files in one repo | Set `arduino-sketch` explicitly |

## Project requirements

| Requirement | Details |
| --- | --- |
| Sketch file | At least one `.ino` file |
| FQBN | Valid board identifier (for example `esp32:esp32:esp32`) |
| Board cores | Installed automatically by `arduino-cli core update-index` in CI |
| firmngin board/model | Device or template fields match your FQBN target |

Example sketch layout:

```text
firmware/
  greenhouse-node/
    greenhouse-node.ino
    config.h
```

The main sketch folder name must match the `.ino` filename (Arduino convention).

## How auto-detection works

When `platform: auto` and no `platformio.ini` exists:

1. The action searches up to **three directory levels** for the first `.ino` file.
2. Sets `platform` to `arduino`.
3. Uses the detected sketch path unless you override with `arduino-sketch`.
4. Requires `arduino-fqbn` — the action fails if FQBN is missing.

<Warning>
  Auto-detection picks the **first** `.ino` found. Repositories with multiple sketches must set `arduino-sketch` explicitly.
</Warning>

## Find your FQBN

Run locally after installing Arduino CLI:

```bash
arduino-cli core update-index
arduino-cli board listall esp32
```

Example output:

```text
Board Name          FQBN
ESP32 Dev Module    esp32:esp32:esp32
```

Use the FQBN value in your workflow:

```yaml
arduino-fqbn: esp32:esp32:esp32
```

Common FQBN values:

| Hardware | Example FQBN |
| --- | --- |
| ESP32 Dev Module | `esp32:esp32:esp32` |
| ESP8266 NodeMCU | `esp8266:esp8266:nodemcuv2` |
| Arduino Uno | `arduino:avr:uno` |

## Basic workflow

```yaml
name: Arduino OTA

on:
  push:
    branches:
      - main

permissions:
  contents: read

jobs:
  ota:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: firmngin/ota@v1.1
        with:
          token: ${{ secrets.FIRMNGIN_API_TOKEN }}
          device-template-id: ${{ vars.FIRMNGIN_DEVICE_TEMPLATE_ID }}
          platform: arduino
          arduino-fqbn: esp32:esp32:esp32
          arduino-sketch: firmware/greenhouse-node/greenhouse-node.ino
          version: ${{ github.run_number }}
```

## Sketch in repository root

If your sketch is at `MyDevice/MyDevice.ino`:

```yaml
- uses: firmngin/ota@v1.1
  with:
    token: ${{ secrets.FIRMNGIN_API_TOKEN }}
    device-secret-id: ${{ vars.FIRMNGIN_DEVICE_SECRET_ID }}
    platform: arduino
    arduino-fqbn: esp32:esp32:esp32
    arduino-sketch: MyDevice/MyDevice.ino
```

When only one `.ino` exists and auto-detection finds it, `arduino-sketch` is optional.

## Including the Firmngin Arduino library

If your sketch depends on the Firmngin Arduino library, install it before the OTA step or vendor it in your repository.

### Option A: Install library in a prior step

```yaml
steps:
  - uses: actions/checkout@v4

  - name: Install Firmngin library
    run: |
      arduino-cli lib install "Firmngin"

  - uses: firmngin/ota@v1.1
    with:
      token: ${{ secrets.FIRMNGIN_API_TOKEN }}
      device-template-id: ${{ vars.FIRMNGIN_DEVICE_TEMPLATE_ID }}
      platform: arduino
      arduino-fqbn: esp32:esp32:esp32
      arduino-sketch: src/device/device.ino
```

<Note>
  The `firmngin/ota` action runs `arduino-cli core update-index` and `arduino-cli compile` internally. Pre-install libraries in separate steps **before** the action if your sketch requires them.
</Note>

### Option B: Vendor library in `lib/` folder

Copy required library sources into your repository under a `lib/` directory and reference them from your sketch. This makes CI builds deterministic without external library downloads.

## Board and model alignment

Map your FQBN target to firmngin board fields:

| FQBN target | `target-board` (artifact-only) | `target-model` (artifact-only) |
| --- | --- | --- |
| `esp32:esp32:esp32` | `esp32` | `esp32` |
| `esp8266:esp8266:nodemcuv2` | `esp8266` | `esp8266` |

For `device-template-id` or `device-secret-id` modes, firmngin resolves board/model from the template or device. Your compiled binary must still be compatible with that hardware.

## Deployment modes

### Fleet OTA via template

```yaml
device-template-id: ${{ vars.FIRMNGIN_DEVICE_TEMPLATE_ID }}
```

Requires a template with **Firmware OTA policy** configured.

### Single device

```yaml
device-secret-id: ${{ vars.FIRMNGIN_DEVICE_SECRET_ID }}
```

Device must have board name and model populated in the dashboard.

### Artifact only

```yaml
target-board: esp32
target-model: esp32
```

Upload without triggering OTA. Deploy manually from the dashboard.

## Version and release notes

```yaml
- uses: firmngin/ota@v1.1
  with:
    token: ${{ secrets.FIRMNGIN_API_TOKEN }}
    device-template-id: ${{ vars.FIRMNGIN_DEVICE_TEMPLATE_ID }}
    platform: arduino
    arduino-fqbn: esp32:esp32:esp32
    version: "2.1.0"
    release-notes: "Fix sensor polling interval; improve MQTT reconnect"
```

If `version` is omitted, the action uses git describe or commit SHA.

## Local verification

Reproduce the CI build locally:

```bash
arduino-cli core update-index
arduino-cli core install esp32:esp32
arduino-cli lib install "Firmngin"
arduino-cli compile --fqbn esp32:esp32:esp32 MyDevice/MyDevice.ino
find MyDevice -name "*.bin"
```

Fix compile and linker errors before pushing to GitHub.

## Binary output location

After compile, the action searches the sketch directory for the first `.bin` file. If compile succeeds but upload fails with "Firmware binary not found":

- Confirm the core actually produces a `.bin` for your board.
- Some boards require export flags — test locally first.

## Multiple sketches in one repository

Use separate jobs or a matrix:

```yaml
strategy:
  matrix:
    sketch:
      - path: products/sensor-a/sensor-a.ino
        fqbn: esp32:esp32:esp32
        template: FIRMNGIN_SENSOR_A_TEMPLATE
      - path: products/sensor-b/sensor-b.ino
        fqbn: esp8266:esp8266:nodemcuv2
        template: FIRMNGIN_SENSOR_B_TEMPLATE
steps:
  - uses: actions/checkout@v4

  - uses: firmngin/ota@v1.1
    with:
      token: ${{ secrets.FIRMNGIN_API_TOKEN }}
      device-template-id: ${{ vars[matrix.template] }}
      platform: arduino
      arduino-fqbn: ${{ matrix.fqbn }}
      arduino-sketch: ${{ matrix.sketch }}
```

## Security reminders

- Add `keys.h` to `.gitignore` — never commit device credentials.
- Store the firmngin API token only in GitHub Secrets.
- Use branch protection on production branches.

## Common Arduino CLI errors

| Error | Cause | Fix |
| --- | --- | --- |
| `arduino-fqbn is required` | Missing FQBN input | Set `arduino-fqbn` in workflow |
| `Unable to detect PlatformIO or Arduino project` | No `.ino` in search path | Add sketch or set `platform: arduino` + `arduino-sketch` |
| Compile error: library not found | Missing dependency | Install library in prior step or vendor in repo |
| Wrong sketch compiled | Multiple `.ino` files | Set `arduino-sketch` explicitly |
| `device board_name is not set` | Single-device mode | Set board fields on device in dashboard |

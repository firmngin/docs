## Overview

This guide walks you through creating an account, registering your first device, and seeing live telemetry in the dashboard.

<Info>
  **Prerequisites**

  - An ESP32 or ESP8266 board
  - Arduino IDE or PlatformIO installed
  - A firmngin.dev account ([sign up here](https://firmngin.dev/register))
</Info>

## Step 1: Create a workspace

<AccordionGroup>
  <Accordion title="Sign up and create workspace" icon="user-plus">
    1. Go to [firmngin.dev/register](https://firmngin.dev/register) and create your account.
    2. Once logged in, create a **workspace** from the dashboard.
    3. Give your workspace a name — this is your project's home for devices, automations, and billing.

    <Tip>
      A workspace isolates all resources: devices, merchants, rules, and API keys.
    </Tip>
  </Accordion>
</AccordionGroup>

## Step 2: Register a device

<AccordionGroup>
  <Accordion title="Add a device to your workspace" icon="microchip">
    1. In the dashboard, navigate to **Devices** and click **Add Device**.
    2. Select your board type (ESP32, ESP8266, etc.).
    3. Provide a device name and optional description.
    4. Once created, you will receive:
       - **Secret ID** — your device identifier
       - **Secret Key** — authentication key
    5. Download the **`keys.h`** file from your device page — this contains your mTLS credentials.

    <Warning>
      Never commit `keys.h` to a public repository. Add it to `.gitignore` to prevent accidental exposure.
    </Warning>
  </Accordion>
</AccordionGroup>

## Step 3: Install the SDK

<AccordionGroup>
  <Accordion title="Arduino IDE" icon="code">
    1. Download the library from [firmngin/Firmngin-Arduino](https://github.com/firmngin/Firmngin-Arduino).
    2. In Arduino IDE, go to **Sketch** → **Include Library** → **Add .ZIP Library...**
    3. Select the downloaded ZIP file.
    4. The library will be added to your Arduino libraries automatically.
  </Accordion>

  <Accordion title="PlatformIO" icon="code">
    Add the following to your `platformio.ini`:

    ```ini
    lib_deps =
      firmngin/Firmngin-Arduino
    ```

    Or use the PlatformIO Library Registry to search for `firmngin`.
  </Accordion>
</AccordionGroup>

## Step 4: Connect your device

Copy and upload this example sketch to your board:

```cpp
#include <Arduino.h>
#include <firmngin.h>

#if defined(ESP8266)
#include <ESP8266WiFi.h>
#elif defined(ESP32)
#include <WiFi.h>
#endif

#define DEVICE_ID   "YOUR_DEVICE_ID"
#define DEVICE_KEY  "YOUR_DEVICE_SECRET_KEY"

const char *ssid = "YOUR_SSID";
const char *password = "YOUR_PASSWORD";

Firmngin fngin(DEVICE_ID, DEVICE_KEY);

Entity temperature("temperature");

void setup() {
  Serial.begin(115200);

  // Connect to WiFi
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\nWiFi connected");

  fngin.setDebug(true);
  fngin.begin();
}

void loop() {
  fngin.loop();

  // Push entity state every 10 seconds
  static unsigned long lastPush = 0;
  if (millis() - lastPush > 10000) {
    lastPush = millis();
    fngin.pushEntity(temperature, "25.5");
  }
}
```

<AccordionGroup>
  <Accordion title="Upload and verify" icon="check">
    1. Select your board and port in Arduino IDE.
    2. Click **Upload**.
    3. Open the **Serial Monitor** at `115200` baud.
    4. You should see WiFi connection messages and device status.
    5. Go to your **Dashboard** — the device should appear as **Online** with live data.
  </Accordion>
</AccordionGroup>

## Step 5: See it live

<AccordionGroup>
  <Accordion title="Real-time dashboard" icon="eye">
    Once your device is connected:

    - **Status** — Online/offline indicator updates in real-time via MQTT LWT
    - **Entities** — GPIO states, sensor readings, and custom entities are synced via `pushEntity()`

    Use `Entity` objects to define typed keys for your data, and `pushEntity()` to send state updates to the dashboard.
  </Accordion>
</AccordionGroup>
